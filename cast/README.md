# Cast: the project's Who

Personas: hypothetical archetypal actors, grounded in research, that keep the work
anchored in who it serves. The process comes from Powell, Keenan and McDaid (2007),
which builds on the personas in Cooper and Reimann's Interaction Design. See
[applying-personas.md](applying-personas.md) for the workshop steps.

This project serves three personas, built by discussion with the operator:

- [Casey](casey.md), a retro gamer who wants authentic Ad Lib Gold sound on
  Windows 9x.
- [Piotr](piotr.md), a driver developer who builds and extends the WDM driver on
  Windows 2000 and XP.
- [Quill](quill.md), a documentation agent that reads the manual and the program
  data and grounds its answers in the source.

The primary persona (the one who must be satisfied and cannot be served by an
interface built for another) is selected with the operator when planning begins,
and it prioritises the milestones in `plan/`.
